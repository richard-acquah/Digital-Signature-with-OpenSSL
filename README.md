# Digital Signature with OpenSSL.
## Objectives
This lab seeks to demonstrate how digital signatures can be generated and used to sign documents. The lab focuses on using Cisco's Skills for all derivatives of Ubuntu Linux to generate private and public keys and use the private key to digitally sign a document.
## Skill Learned
+ Application of digital signature.
## Resources / Tools /Utilities
+ Cisco's CSE-LABVM VM(Ubuntu derivative)
+ Open SSL
## Steps 1
To generate a private key, open the terminal and enter the command: ***openssl genpkey -algorithm RSA -out private_key.pem***:

Verify that the private key is generated by displaying the content of private_key.pem using the ***cat*** command:

<img width="947" alt="Privatekeyverify" src="https://github.com/richard-acquah/Digital-Signature-with-OpenSSL/assets/136107996/1e6b67d4-efac-4f1f-9731-f0607e85ce92">

## Step 2
To generate a public key, use the command: ***openssl pkey*** ___-in private_key.pem -pubout -out public_key.pem___.

Again, Verify that the public key is generated by displaying the content of public_key.pem on the terminal:

<img width="947" alt="publickeygenverify" src="https://github.com/richard-acquah/Digital-Signature-with-OpenSSL/assets/136107996/bcfe08cc-b7cb-4a58-b6d3-ae1543608b40">

## Step 3
Create a document that is going to be signed with the private key. The ***echo*** is used to create the document.

Verify the content of confidential.txt by displaying it on the terminal:

<img width="949" alt="echodocum" src="https://github.com/richard-acquah/Digital-Signature-with-OpenSSL/assets/136107996/e5a6f830-fe4e-45c1-acf1-1e1986b41187">

## Step 4 
To sign the document with the private key, the command ***openssl dgst -sha256 -sign private_key.pem -out signature confidential.txt***.

Display the content of the signature using the cat on the terminal.

<img width="949" alt="signing docu and verify" src="https://github.com/richard-acquah/Digital-Signature-with-OpenSSL/assets/136107996/bc361923-a0fd-429c-bd37-6723a002dff9">

## Step 5
To verify the authenticity and integrity of the document, the command  ***openssl dgst -sha256 -verify public_key.pem -signature signature confidential.txt*** is used:


The document's authenticity and integrity are verified by printing **Verified OK** on the terminal.
